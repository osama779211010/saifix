# توثيق API شبكات الحوالات (Remittance Networks API)

يوفر هذا الـ API قائمة بجميع شبكات الحوالات النشطة المعدة في النظام، مع كافة التفاصيل اللازمة لمبرمج التطبيق (Mobile App) لعرض القائمة والتعامل مع روابط الـ API الخاصة بكل شبكة.

## معلومات الـ Endpoint الأساسية

- **URL:** `/api/settings/remittance-networks/`
- **Method:** `GET`
- **Authentication:** `Token` أو `Bearer` (يجب أن يكون المستخدم مسجلاً دخوله)
- **Status:** نشط (يعيد فقط الشبكات التي تكون `is_active=true`)

---

## هيكلية البيانات (JSON Response)

يعيد الـ API مصفوفة (Array) من الكائنات، مرتبة حسب حقل `sort_order` الذي يحدده المدير من لوحة التحكم.

### مثال على الرد (Response Example):
```json
[
  {
    "id": 1,
    "name_ar": "شبكة النجم للحوالات",
    "name_en": "Al-Najm Remittance",
    "network_code": "ALNAJM",
    "sort_order": 1,
    "api_key": "sk_test_51Mz...",
    "endpoint_url": "https://api.alnajm.com/v1/",
    "is_active": true,
    "icon": "http://server-ip/media/settings/networks/icons/alnajm.png",
    "created_at": "2024-05-11T16:00:00Z",
    "updated_at": "2024-05-11T16:30:00Z"
  }
]
```

## شرح الحقول لمبرمج التطبيق

| الحقل | النوع | الوصف | ملاحظات للمبرمج |
| :--- | :--- | :--- | :--- |
| `id` | Integer | المعرف الفريد للشبكة | يستخدم في العمليات الداخلية |
| `name_ar` | String | اسم الشبكة بالعربي | يعرض في واجهة اللغة العربية |
| `name_en` | String | اسم الشبكة بالإنجليزي | يعرض في واجهة اللغة الإنجليزية |
| `network_code` | String | رمز الشبكة (Short Code) | هام جداً عند إرسال طلبات الحوالة لتحديد الشبكة |
| `sort_order` | Integer | ترتيب العرض | يجب ترتيب القائمة في التطبيق تصاعدياً حسب هذا الحقل |
| `api_key` | String | مفتاح الوصول للشبكة | قد تحتاجه إذا كان التطبيق يقوم بطلبات مباشرة (تأكد من أمن البيانات) |
| `endpoint_url` | String | رابط الـ API للشبكة | الرابط الذي يتم توجيه الطلبات إليه لهذه الشبكة تحديداً |
| `icon` | String (URL) | رابط أيقونة الشبكة | رابط كامل للصورة، يمكن استخدامه في `Image.network` أو ما يعادلها |
| `is_active` | Boolean | حالة الشبكة | الـ API يعيد فقط النشطة، لكن الحقل موجود للتأكيد |

---

## تعليمات التكامل لمبرمج التطبيق

1. **الترتيب (Sorting):** لا تعتمد على ترتيب الـ Array القادم، بل تأكد دائماً من استخدام حقل `sort_order` لترتيب القائمة أمام المستخدم.
2. **الأيقونات (Icons):** الحقل يعيد رابطاً كاملاً. يفضل عمل Cache للصور في التطبيق لتقليل استهلاك البيانات.
3. **التحديث (Refresh):** يفضل جلب هذه البيانات عند تشغيل التطبيق أو الدخول لصفحة الحوالات لضمان حصول المستخدم على أحدث الإعدادات (مثل توقف شبكة معينة أو تغيير رابط).
4. **الأمان:** حقل `api_key` يعتبر بيانات حساسة، يفضل عدم تخزينه بشكل دائم في الـ local storage الخاص بالتطبيق إلا إذا كان ضرورياً للتشغيل.
