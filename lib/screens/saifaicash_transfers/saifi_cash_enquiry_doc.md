# توثيق API الاستعلام عن حالة الحوالة (Saifi Cash Enquiry)

هذا الـ API مخصص لتطبيقات الجوال (Flutter) لتمكين المستخدم من معرفة حالة الحوالة التي قام بإرسالها مسبقاً (هل استُلمت أم لا).

---

## المعلومات الأساسية
- **المسار (Endpoint):** `/api/SaifiCash/wallet/enquire-status/`
- **الطريقة (Method):** `POST`
- **التوثيق (Auth):** يتطلب `Bearer Token` (مستخدم مسجل دخول).
- **نوع المحتوى (Content-Type):** `application/json`

---

## 1. المعاملات المرسلة (Request Body)

يجب إرسال كائن JSON يحتوي على رقم الحوالة:

| الحقل | النوع | الوصف | مثال |
| :--- | :--- | :--- | :--- |
| `rmt_no` | String | (إجباري) رقم الحوالة الذي حصل عليه المستخدم عند الإرسال | `"AGENT-123456"` |

---

## 2. مخرجات النجاح (Success Response)

يعيد السيرفر كائن JSON يوضح الحالة الحالية للحوالة في نظام "صيفي كاش".

### مثال للرد (الحوالة مستلمة):
```json
{
    "status": "success",
    "message": "تم الاستعلام بنجاح. الحالة الحالية: paid",
    "rmt_status": "paid"
}
```

### قيم حقل `rmt_status` المتوقعة:
- `unpaid`: الحوالة لا تزال قائمة ولم يتم استلامها بعد.
- `paid`: تم استلام الحوالة بنجاح من قبل المستفيد.
- `notfound`: الحوالة غير موجودة في نظام المزود.

---

## 3. رسائل الخطأ (Error Responses)

| الحالة | الكود | JSON المرتجع | السبب |
| :--- | :--- | :--- | :--- |
| **خطأ مدخلات** | 400 | `{"error": "يجب إدخال رقم الحوالة"}` | لم يتم إرسال حقل `rmt_no` |
| **غير موجودة** | 404 | `{"error": "الحوالة غير موجودة أو غير مصرح لك..."}` | الرقم خاطئ أو لا يخص المستخدم الحالي |
| **خطأ مزود الخدمة** | 400 | `{"status": "error", "message": "..."}` | مشكلة في الاتصال بسيرفرات صيفي كاش |

---

## ملاحظة لمبرمج الـ Flutter:
- تأكد من تمرير `Authorization: Bearer <Token>` في الـ Header.
- يفضل عرض حالة الـ `rmt_status` للمستخدم بشكل واضح (مثلاً لون أخضر لـ `paid` ولون برتقالي لـ `unpaid`).
