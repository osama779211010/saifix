# توثيق واجهة تسجيل الدخول فائقة السرعة (Fast Wallet Login API)

هذه الواجهة مصممة لتوفير أداء فائق السرعة (أقل من 0.02 ثانية) ولدعم تسجيل الدخول المزدوج (مستخدم عادي أو نقطة مبيعات).

---

## المعلومات الأساسية
- **المسار (Endpoint):** `/api/authentication/fast-login/`
- **الطريقة (Method):** `POST`
- **نوع المحتوى (Content-Type):** `application/json`

---

## 1. المعاملات المرسلة (Request Body)

يجب إرسال كائن JSON يحتوي على الحقول التالية:

| الحقل | النوع | الوصف | مثال |
| :--- | :--- | :--- | :--- |
| `username` | String | (إجباري) رقم الهاتف أو رقم نقطة المبيعات | `"777123456"` |
| `password` | String | (إجباري) كلمة المرور الخاصة بالحساب | `"P@ssw0rd123"` |
| `user_type` | String | (إجباري) نوع الدخول: `phone_number` أو `pos_point` | `"phone_number"` |
| `mac_address` | String | (اختياري) عنوان الماك للجهاز | `"00:1A:2B:3C:4D:5E"` |

---

## 2. مخرجات النجاح (Success Response)

يعيد السيرفر كائن JSON يحتوي على التوكنات (Tokens) والبروفايل المختصر للمستخدم.

---

## 3. رسائل الخطأ (Error Responses)

| الحالة | الكود | JSON المرتجع | السبب |
| :--- | :--- | :--- | :--- |
| **خطأ مدخلات** | 400 | `{"error": "يجب إدخال اسم المستخدم وكلمة المرور"}` | عدم إرسال أحد الحقول الإجبارية |
| **خطأ مصادقة** | 401 | `{"error": "بيانات الدخول غير صحيحة"}` | كلمة مرور خاطئة أو مستخدم غير موجود |
| **نقطة غير نشطة** | 401 | `{"error": "نقطة المبيعات غير موجودة أو غير نشطة"}` | رقم نقطة خاطئ أو تم إيقافها |
| **خطأ سيرفر** | 500 | `{"error": "حدث خطأ غير متوقع: ..."}` | حدوث استثناء في الكود البرمجي |

---

## ملاحظات تقنية للمبرمج:
1. **عنوان الماك (MAC Address):** يتم إرساله مرة واحدة في أول دخول لتثبيته في السيرفر.
2. **الأرصدة (Wallets):** مصفوفة تحتوي على أرصدة جميع العملات المتاحة للمستخدم مباشرة في رد تسجيل الدخول.
3. **نوع الدخول:** تأكد من إرسال `user_type` بدقة ليقوم السيرفر بتوجيه الطلب للمسار الصحيح.
